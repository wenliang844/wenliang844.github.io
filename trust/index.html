隐私与信任

本站如何处理数据

这里集中说明哪些数据留在你的浏览器、哪些第三方服务可能被连接，以及你可以怎样清理或控制本机数据。

0 内置私密 API key
本机优先 工具与助手默认在浏览器内处理
可清理 本机历史与草稿由用户控制
本机优先

保存在本机的数据

站点会用浏览器存储支持便捷功能；这些记录限定在当前浏览器配置中，可由你清理。

AI 助手设置与对话

localStorage: cwl.assistant.*

站内规则模式不会外发对话；启用大模型模式时，API key、模型和中转站配置保存在当前浏览器。隐私模式会阻止对话历史持久化。

可在助手面板清空全部对话、切换隐私模式或调整保留期限。

API Tester 历史

localStorage: cwl.tools.apiHistory

历史仅用于快速复用请求；保存前会脱敏敏感 Header，请求体默认不保存，除非用户主动勾选保存。

可在工具面板删除单条历史或清空全部历史。

反馈、草稿与阅读偏好

localStorage / sessionStorage

留言反馈默认保存在当前浏览器；Markdown、简历模板、阅读进度、主题和语言偏好也保存在本机。

可在对应页面删除反馈、重置草稿，或通过浏览器站点数据面板统一清除。

用户控制

你可以控制什么

  • 清空本机数据

    可以在 AI 助手、API Tester、反馈页和编辑器中删除对应记录；也可以在浏览器的站点设置中清除本站全部数据。

  • 外发请求由用户动作触发

    订阅、评论、赞助、在线反馈、AI 大模型请求和 API Tester 请求都需要用户主动操作或配置后才会发生。

  • 客户端 secret 不等于真正保密

    任何填在网页里的 API key 都只能保存在当前浏览器，不能当作服务端密钥使用；建议为站点单独创建可撤销 key。

安全摘要

安全说明

  • 静态站点默认无自有后端

    公开页面由 GitHub Pages 托管；默认没有自建服务器接收访问者表单或工具输入。

  • CSP 限制资源来源

    页面声明 Content Security Policy，限制脚本、样式、图片、连接、表单和 iframe 的允许来源。

  • 安全问题反馈入口

    如发现泄露、误导性数据流或安全问题，请通过联系页反馈，维护者会优先处理。

反馈安全或隐私问题

如果这里的说明不准确，或你发现安全/隐私问题，请通过联系页留言。
